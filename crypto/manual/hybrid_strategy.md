# Гибридная модель: Student-t + GBM

## Контекст

До этого момента бот использовал единственную модель — **Student-t** (fat tails, df=2.61 BTC / 2.88 ETH) — для всех рынков. Бэктест на 66,620 синтетических наблюдениях (июнь 2023 — март 2026) показал, что ни Student-t, ни GBM не являются лучшей моделью *во всех* ситуациях. Оптимальная стратегия — **гибрид**, который использует разные модели для разных типов рынков.

| Горизонты | 30, 90, 180, 365 дней | — |
| **Итого наблюдений** | **66,620** | — |

### Методика

Для каждого торгового дня, для каждой комбинации (валюта × страйк × горизонт):
1. Вычисляем touch probability по 4 моделям (Student-t d=0, GBM d=0, Student-t+drift, GBM+drift)
2. Проверяем реальность: коснулась ли цена страйка за указанный горизонт (binary: 0 или 1)
3. Считаем Brier score = mean((predicted - actual)²) — чем меньше, тем лучше
4. Drift для каждого дня рассчитывается из ближайшего квартального фьючерса (30-180 DTE)

### Главная таблица: 5 моделей

| Модель | Brier | Bias | Описание |
|--------|-------|------|----------|
| Student-t d=0 | 0.18295 | -0.058 | Fat tails, drift=0 |
| GBM d=0 | 0.18476 | +0.016 | Normal, drift=0 |
| Student-t+drift | 0.17659 | -0.064 | Fat tails + futures drift |
| GBM+drift | 0.17419 | +0.006 | Normal + futures drift |
| **Hybrid+drift** | **0.16707** | **-0.028** | **St-t для dip + GBM для reach + drift** |

**Hybrid выигрывает у всех моделей** с Brier 0.167 vs следующий лучший 0.174.

## Почему так?

### Student-t для dip (падения)

Student-t распределение с df=2.61 моделирует **fat tails** — экстремальные дневные движения случаются в ~5-6 раз чаще, чем по Normal. Для dip-рынков это означает:

- Student-t лучше моделирует **"застревание" цены** вблизи текущего уровня (высокий пик в центре распределения)
- Меньше средних шагов → цена реже "продвигается" к далёкому нижнему страйку
- Огромные шаги случаются, но в случайную сторону — они не помогают систематически достичь нижнего барьера
- **Результат:** Student-t даёт более низкую touch probability для dip, что ближе к реальности

GBM переоценивает dip probability (bias +0.108 для dip), потому что нормальное распределение даёт слишком много средних шагов, которые суммируясь "доводят" цену до нижнего барьера.

### GBM для reach (роста)

Для reach-рынков Student-t **недооценивает** вероятность касания (bias -0.169 для reach). Почему:

- Криптовалюты имеют **положительный drift** (фьючерсная кривая в контанго, mean ~7.5%/yr для BTC)
- Рост BTC часто происходит через **последовательные средние шаги** (постепенный тренд вверх), а не через один огромный прыжок
- GBM с нормальным распределением лучше моделирует этот "трендовый" рост
- Student-t "топчется на месте" (высокий пик) и изредка делает огромные шаги — но для reach нужен стабильный апсайд

### ETH: та же картина

Паттерн "Student-t для dip, GBM для reach" сохраняется для ETH без изменений:
- ETH dip: Student-t Brier 0.14848 vs GBM 0.15209
- ETH reach: GBM Brier 0.21885 vs Student-t 0.23352

Отдельная модель для ETH не нужна — гибрид универсален.

## Влияние drift

Drift из фьючерсной кривой **улучшает обе модели**, но GBM получает бо́льший буст:

| Модель | Без drift | С drift | Улучшение |
|--------|-----------|---------|-----------|
| Student-t | 0.18295 | 0.17659 | -3.5% |
| GBM | 0.18476 | 0.17419 | -5.7% |

GBM сильнее реагирует на drift, потому что нормальное распределение "чисто" сдвигает среднее, а у Student-t толстые хвосты частично "размывают" эффект сдвига.

Drift по кварталам (BTC):
- 2023-Q1: +2.3%/yr (восстановление после краша)
- 2024-Q1: +14.6%/yr (halving rally)
- 2024-Q4: +12.1%/yr (election + bull run)
- 2026-Q1: +3.5%/yr (текущий, сдержанный)

## Гибридная модель: реализация

### Правило (простое и устойчивое)

```
if strike < spot:      # dip market
    touch_prob = student_t_touch_prob(spot, strike, iv, T, drift, df)
else:                  # reach market
    touch_prob = gbm_touch_prob(spot, strike, iv, T, drift)
```

### Изменения в коде

**Файл:** `crypto/trading_bot/pricing/fast_approx.py`

Текущая функция `fast_touch_prob()` всегда применяет Student-t коррекцию (lookup table). Нужно добавить параметр `hybrid=True`:

```python
def fast_touch_prob(spot, strike, iv, T, drift=0.0, df=2.61, hybrid=False):
    is_up = strike > spot

    # GBM аналитически
    if is_up:
        gbm = touch_above_gbm(spot, strike, iv, T, mu=drift)
    else:
        gbm = touch_below_gbm(spot, strike, iv, T, mu=drift)

    if hybrid and is_up:
        # Reach: GBM без коррекции
        return gbm

    # Dip (или non-hybrid): Student-t = GBM × correction
    correction = _lookup_correction(spot, strike, iv, T, df)
    return min(1.0, max(0.0, gbm * correction))
```

### Где менять

| Файл | Что меняется |
|------|-------------|
| `fast_approx.py` | Добавить `hybrid` параметр в `fast_touch_prob()` и `batch_fast_touch_probabilities()` |
| `scanner/crypto.py` | Передавать `hybrid=True` при вызове batch pricing |
| `full_scan.py` | Добавить колонку "Hybrid edge" в отчёт |
| `strategy.ipynb` | Показывать hybrid edge рядом с текущими |

### Влияние на edge

Для **dip-рынков** (наша основная позиция BTC dip $55k NO): edge не меняется — Student-t остаётся.

Для **reach-рынков** edge изменится:
- Student-t даёт МЕНЬШУЮ touch prob → завышает edge на NO / занижает на YES
- GBM даёт БОЛЬШУЮ touch prob → более агрессивный edge на YES
- Пример: BTC reach $120k, Student-t: YES fair=18¢, GBM: YES fair=22¢ → hybrid берёт GBM (22¢)

## Ограничения бэктеста

1. **DVOL доступен только с июня 2023.** Нет данных за bear market 2022 (крах FTX, Luna). Режим "bear" в бэктесте — это снижения 2023-2025 (мягкие по сравнению с 2022).

2. **Drift из квартальных фьючерсов.** В реальной торговле бот подбирает drift по сроку рынка (Mar рынки → Mar futures, Annual → Dec futures). Бэктест использует "ближайший 30-180 DTE" фьючерс — это огрубление.

3. **Synthetic markets.** Реальные Polymarket рынки могут иметь нестандартные страйки и сроки. Synthetic markets покрывают ±10-50% от спота и 30-365 дней — это хорошее приближение.

4. **Overlapping observations.** Наблюдения для одного дня с разными горизонтами пересекаются. Это не влияет на bias, но может занижать variance оценки Brier score.

5. **No transaction costs.** Brier score оценивает калибровку модели, не прибыльность. Прибыльность зависит от Polymarket spread, ликвидности и timing.
