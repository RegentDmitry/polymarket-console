# Hedging Plan — когда депозит вырастет до $5-10k+

## Текущие лимиты (работают для $1-2k)
- max_position_pct: 25%
- max_direction_pct: 60%
- min_position_size: $5

## Что реализовать

### 1. Direction exposure tracking
Считать net direction exposure корректно:
- **Бычий** (BTC не упадёт / вырастет): dip NO, reach YES
- **Медвежий** (BTC упадёт / не вырастет): dip YES, reach NO
- Показывать в TUI: "Bull: 85% | Bear: 15%"

### 2. Новые параметры конфига
```python
max_direction_pct: float = 0.50   # макс 50% в одном направлении (сейчас 60%)
max_asset_pct: float = 0.70       # макс 70% в одном активе (BTC/ETH)
min_positions: int = 5            # мин количество позиций (диверсификация)
```

### 3. Логика хеджирования в сканере
- Если direction > 50% → не покупать больше в этом направлении
- Если direction > 50% И есть edge на противоположной стороне → приоритет хеджу
- Если portfolio < min_positions → снизить min_edge для новых рынков (но не ниже 3%)

### 4. Кросс-актив диверсификация
- Макс 70% портфеля в одном активе (BTC)
- ETH как частичная диверсификация (корреляция ~0.85, слабый хедж)
- Политика — некоррелированный хедж (лучший вариант)

### 5. Предложенные лимиты для $5-10k
| Параметр | $1-2k | $5k+ | $10k+ |
|----------|-------|------|-------|
| max_position_pct | 25% | 15% | 10% |
| max_direction_pct | 60% | 50% | 40% |
| max_asset_pct | нет | 70% | 60% |
| min_positions | нет | 5 | 8 |

### 6. Парный трейд (advanced)
Настоящий хедж на Polymarket дорогой (рынок эффективный). Лучшая стратегия:
- Диверсификация по некоррелированным рынкам > покупка противоположных страйков
- dip NO + dip YES (разные страйки) работает только если оба имеют edge
- Не покупать хедж с отрицательным edge — лучше просто уменьшить позицию
