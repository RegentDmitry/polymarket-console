## Шаг 2b: Фьючерсные кривые BTC и ETH (ОБЯЗАТЕЛЬНО, ПЕРЕГЕНЕРИРОВАТЬ КАЖДЫЙ РАЗ)

Фьючерсные кривые **обновляются каждый день** — их нужно получать заново при каждом отчёте. Запусти:

```bash
.venv/bin/python crypto/full_scan.py --futures-only
```

Скрипт `full_scan.py` автоматически:
1. Загружает фьючерсные кривые **BTC и ETH** с Deribit
2. Рассчитывает implied drift для каждого срока (Feb, Mar, Jun, Sep, Dec)
3. При полном скане (без `--futures-only`) **подбирает drift по сроку рынка** — Feb рынки используют Feb futures drift, Annual — Dec futures drift

**Две колонки edge:**
- **MC d=0** — Monte Carlo Student-t, drift=0 (risk-neutral)
- **MC fut** — Monte Carlo Student-t, drift из ближайшего фьючерса по сроку

Добавь в заголовок отчёта обе кривые:
```
BTC futures: Feb $XX,XXX (drift X.X%), ... Dec $XX,XXX (drift X.X%)
ETH futures: Feb $X,XXX (drift X.X%), ... Dec $X,XXX (drift X.X%)
```

### 5b. Крипто — анализ как в боте (hybrid model)


Это выдаст таблицу всех рынков с колонками: MC d=0, MC fut, Hybrid edge.
**Hybrid = основная метрика** (Student-t для dip, GBM для reach + futures drift).

Для каждой крипто позиции из API (Шаг 0b) найди соответствующий рынок в скане и определи:
- **Fair price** = PM price + hybrid edge (для нашей стороны)
- **Edge** = hybrid edge (fair - current)
- Если edge < 0 → позиция потеряла edge, рассмотреть выход (edge_exit)

При текущей цене BTC/ETH также проверь триггеры из exit_strategy.md:

- BTC >$85k → основная фиксация (какие позиции?)
- BTC <$60k → стоп-лосс dip55k NO
- Временные стопы (май <$65k, июль <$75k, октябрь <$90k)

### 5c. Правило SM (обновлено после бэктеста 2026-02-17)

**БЭКТЕСТ ПОКАЗАЛ: SM reversal — ПЛОХОЙ сигнал для выхода.** На 24 политических рынках exit по SM reversal даёт win rate ~40-45% (хуже coin flip), P&L -$20 vs +$1,002 при hold to resolution. SM flow шумный на 5-дневных интервалах — разворот ≠ неправильное направление.

**Правильное использование SM:**
- **ВХОД:** SM flow — хороший фильтр (hit rate 83-94%). Покупай только когда |flow| > 0.1 подтверждает направление.
- **ВЫХОД:** SM reversal НЕ является sell trigger. Hold to resolution лучше на политических рынках.
- **Исключение:** Если SM СИЛЬНО против (|flow| > 0.3 в противоположную сторону) + новости подтверждают → рассмотри выход.

Для каждой **политической** позиции проверь SM flow, но **не продавай только из-за разворота SM:**
- SM подтверждает → 🟢 дополнительная уверенность
- SM нейтрален (|flow| < 0.1) → 🟡 нет сигнала, без действий
- SM сильно против (|flow| > 0.3) → 🟡 watch, проверь новости, НО не автоматический sell

**Для крипто SM используется как дополнительный сигнал** наряду с Deribit touch probability.

> Подробные результаты: `politics/backtest/RESULTS.md`

### 5c-2. Правило Take-Profit (tail risk, обновлено 2026-02-21)

**БЭКТЕСТ НА 181 ПОЛИТИЧЕСКОМ РЫНКЕ:** tail risk реален. Рынки достигавшие 95%+ иногда резолвятся в противоположную сторону. Оценка flip rate: 5-15% (неточно из-за bias в данных).

**Правила take-profit (ТОЛЬКО ДЛЯ ПОЛИТИКИ):**
- **90%+** → рассмотреть partial exit (продать 50%)
- **95%+** → продать всё (если до resolution не <24ч)
- **97%+** → ВСЕГДА продать (EV hold < 3¢, risk > 97¢)
- **Исключение:** до resolution < 24ч и рынок стабилен → hold

**КРИПТО — tail risk НЕ ПРИМЕНЯЕТСЯ.** Крипто price markets (BTC dip/reach) принципиально отличаются от политических:
- Цена BTC наблюдается в реальном времени — можно выйти при приближении к страйку
- Нет "resolution surprise" (формулировки, юридические нюансы, неожиданные решения)
- Deribit touch probability даёт точную оценку риска
- **Правило:** для крипто позиций на 90%+ НЕ генерировать tail risk алерты. Держать до resolution. Единственный sell trigger — приближение BTC к страйку (мониторинг цены).

Для каждой ПОЛИТИЧЕСКОЙ позиции проверь, не достигла ли она уровня take-profit. Для крипто — игнорировать tail risk.

> Подробные результаты: `politics/backtest/RESULTS.md` (Фаза 3)

### 5d. Анализ формулировок резолюции (ОБЯЗАТЕЛЬНО)

**КРИТИЧЕСКИ ВАЖНЫЙ ШАГ.** Для КАЖДОЙ позиции проверь точные критерии резолюции через Gamma API:

```bash
curl -s "https://gamma-api.polymarket.com/events?slug=<event-slug>" | python3 -m json.tool
```

Для каждого рынка ищи СКРЫТЫЕ НЮАНСЫ в формулировках:

1. **"Новый" vs "продолжение"** — рынок может требовать НОВОЕ событие, а не продолжение существующего (пример: shutdown market требовал "новый" шатдаун, продолжение Jan 31 не считалось → потеряли $145)
2. **Точное время дедлайна** — "by February 14" может означать 23:59 ET 14-го, а событие начинается 15-го → не считается
3. **Источник резолюции** — какой именно орган/сайт используется для определения? (OPM, Reuters, AP, etc.)
4. **Определение терминов** — что считается "acquire" (покупка? аренда? военная база?), "shutdown" (полный? частичный? DHS-only?), "strike" (ракетный? кибер? sanctions?)
5. **Исключения** — что явно НЕ считается (погода, продолжение, технический дефолт)
6. **Суб-рынки** — убедись что смотришь правильный суб-рынок (Feb 28 vs Mar 31, etc.)

**ФОРМАТ ВЫВОДА** — для каждой позиции добавь в отчёт:
```
⚖️ Резолюция: [краткое описание критериев]
⚠️ Скрытые нюансы: [что может пойти не так из-за формулировки]
```

> **Урок из Gov't Shutdown:** Мы купили YES @ 94¢ думая что DHS partial shutdown = government shutdown. Оказалось, рынок требовал "новый" шатдаун (не продолжение Jan 31), и дедлайн 14 фев 23:59 ET мог не захватить шатдаун начавшийся 15-го. Потеря: -$145. ВСЕГДА читай мелкий шрифт.

### 5e. Крипто: полный скан + ротация (hybrid model)

**Основной инструмент:** `full_scan.py`. Методология:
- **Hybrid model**: Student-t для dip (fat tails), GBM для reach (normal tails)
- **IV из Deribit** ATM опционов
- **Drift из фьючерсной кривой** (подбирается по сроку рынка)
- **Student-t df**: BTC=2.61, ETH=2.88

**Колонка "Hybrid" — ОСНОВНАЯ метрика.** Это тот же алгоритм что в крипто-боте (`fast_touch_prob` с `hybrid=True`).

**ВАЖНО: все edge — ДЛЯ НАШЕЙ СТОРОНЫ (YES или NO).** `mc_edge()` уже считает для правильной стороны.

`full_scan.py` сканирует ВСЕ активные рынки (не только наши позиции) через Gamma API.

Для ротации:
1. Записать **Hybrid edge** для каждой текущей позиции
2. Найти лучшие возможности среди ВСЕХ рынков
3. Если лучшая возможность имеет edge на **3%+ выше** чем худшая текущая позиция → рекомендовать ротацию

**ФОРМАТ ВЫВОДА:**
```
🔄 РОТАЦИЯ ПОЗИЦИЙ
──────────────────────────────────────
Текущий портфель (по edge):
  1. [позиция] — edge: XX%
  2. [позиция] — edge: XX%

Лучшие доступные (не в портфеле):
  1. [рынок] — edge: XX% [← РОТАЦИЯ если edge > worst + 3%]

Рекомендация: [ПРОДАТЬ worst / КУПИТЬ best] или [Без ротации]
```
